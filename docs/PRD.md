# 任務管理系統 PRD

## 1. 專案概述

本系統是一個簡單的任務管理 Web 應用程式，使用者可以新增、查看、編輯、刪除任務，並標記任務是否完成。系統目標是幫助使用者整理日常待辦事項，提高學習與工作的效率。

## 2. 目標用戶

- 學生
- 上班族
- 需要管理日常待辦事項的人
- 需要記錄學習或工作進度的人

## 3. 核心功能

1. 新增任務
2. 查看任務列表
3. 編輯任務
4. 刪除任務
5. 標記任務完成或未完成
6. 依完成狀態篩選任務

## 4. MVP 範圍

本次課堂練習的最小可行產品包含：

- 使用 Flask 建立後端
- 使用 Jinja2 顯示頁面
- 使用 SQLite 儲存任務資料
- 可以新增、查看、編輯、刪除任務
- 可以標記任務完成狀態
- 可以依照全部、已完成、未完成來篩選任務

## 5. 使用者故事

- 作為一位學生，我希望可以新增任務，方便記錄作業與報告。
- 作為一位使用者，我希望可以查看所有任務，知道目前有哪些事情要做。
- 作為一位使用者，我希望可以編輯任務，修正任務內容。
- 作為一位使用者，我希望可以刪除任務，移除不需要的項目。
- 作為一位使用者，我希望可以標記任務完成，方便追蹤進度。
- 作為一位使用者，我希望可以篩選任務狀態，快速查看未完成的工作。

## 6. 驗收標準

- 使用者可以成功新增任務。
- 使用者可以在首頁看到任務列表。
- 使用者可以編輯既有任務。
- 使用者可以刪除任務。
- 使用者可以切換任務完成狀態。
- 使用者可以依照全部、已完成、未完成篩選任務。
- 任務資料可以正確儲存在 SQLite 資料庫中。
